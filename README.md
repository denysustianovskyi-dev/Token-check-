# Crypto Safety Bot 🛡️

Telegram-бот, який перевіряє безпеку в крипті:
- **/token** — чи не є токен скамом/хаунипотом (податки, мінтинг, верифікація коду, холдери)
- **/wallet** — чи немає у гаманця небезпечних token approvals (дозволів для смартконтрактів)

Дані беруться з безкоштовного публічного [GoPlus Security API](https://gopluslabs.io/) —
не потрібен свій блокчейн-нод чи платні API-ключі для старту.

## Швидкий старт

1. Встановіть залежності:
   ```bash
   pip install -r requirements.txt
   ```

2. Отримайте токен бота у [@BotFather](https://t.me/BotFather) в Telegram.

3. Скопіюйте `.env.example` в `.env` і вставте токен:
   ```bash
   cp .env.example .env
   ```

4. Запустіть бота:
   ```bash
   python bot.py
   ```

5. Напишіть боту `/start` в Telegram.

## Структура проєкту

```
crypto-safety-bot/
├── bot.py                  # точка входу
├── config.py                # налаштування, мережі
├── handlers/
│   ├── start.py              # /start, /help
│   ├── token_check.py        # /token
│   └── wallet_check.py       # /wallet
├── services/
│   ├── token_security.py      # запити до GoPlus (безпека токена)
│   └── approvals.py           # запити до GoPlus (approvals гаманця)
├── requirements.txt
└── .env.example
```

## Важливо перед продакшн-запуском

- **Перевірте актуальну документацію GoPlus API** — https://docs.gopluslabs.io/reference/
  Ендпоінти й назви параметрів іноді змінюються, у коді можуть знадобитись невеликі правки.
- **Рейт-ліміти**: безкоштовний тір GoPlus має обмеження запитів/хвилину. Для більшого
  навантаження зареєструйте app_key/app_secret на їхньому сайті.
- Додайте **обробку помилок і логування** перед публічним запуском (наприклад, Sentry).
- Розгорніть бота на сервері з постійним аптаймом (VPS, Railway, Render) — на своєму
  комп'ютері бот працюватиме, лише поки увімкнений скрипт.

## План монетизації (нагадування)

1. **Тижні 1-4**: повністю безкоштовно, збір користувачів і фідбек у крипто-чатах/Twitter
2. **Далі**: базова перевірка безкоштовна, преміум ($5-10/міс) — автоматичний моніторинг
   гаманця з push-алертами при нових підозрілих approvals (потрібен фоновий воркер,
   що періодично опитує API й порівнює з попереднім станом — зберігайте стан у SQLite/Postgres)
3. Партнерська комісія за перехід на revoke.cash або подібні сервіси, якщо з'явиться
   афіліат-програма

## Наступні кроки для розширення

- Додати базу даних (SQLite) для збереження підписок і історії перевірок користувача
- Фоновий планувальник (APScheduler) для преміум-моніторингу гаманців
- Telegram Stars або криптоплатежі (USDT) для оплати підписки
- Інлайн-кнопки замість команд для зручнішого UX
